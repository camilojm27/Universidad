 TALLER 1

Revise el programa que se le entrega, busque los defectos que tenga y corríjalos. Al final, entregue el programa corregido por medio del campus virtual.

METODOLOGÍA: PAIR PROGRAMMING. En grupos de 2 personas {A,B}, con un solo computador prendido para cada grupo. La persona A maneja el computador, mientras la persona B no puede tocarlo, sino que se limita a dar orientaciones a la persona A. Cada 10 minutos se intercambian los roles.

PREMIO: Tiene toda la semana para realizar este taller, pero si termina durante la clase (y todo está bien) entonces se le sumará 0.1 en la nota final.

RESPECTO AL PROGRAMA (el profesor va a medir cada uno de estos items):

- Estilo:
+ ¿Están en archivos separados el programa principal, las declaraciones de cada clase y las implementaciones de cada clase?
+ ¿Los archivos .h tienen los comentarios de Clase, Responsabilidad y Colaboración?
+ ¿En el programa principal hay un comentario que cuenta una historia?
+ ¿Tienen todos los archivos las cabeceras exigidas (autor, email, fecha)?
+ ¿En cada línea hay una sola sentencia?
+ ¿Están alineadas las llaves en la misma vertical?
+ ¿El código dentro de las llaves está indentado 2 espacios hacia la derecha?
+ ¿Los nombres de las variables, clases, objetos y funciones se entienden?

- Programación:
+ ¿Existe el main y está correctamente declarado?
+ ¿Existen las guardas en los archivos .h y son correctas?
+ ¿Están puestos los #include necesarios?
+ ¿Están las llaves{} adecuadamente puestas para agrupar código?
+ ¿El algoritmo está correctamente implementado?

- Compilación:
+ ¿Compila sin errores?
+ ¿Compila sin warnings? En caso de que haya warnings ¿se pueden eliminar?

- Ejecución:
+ ¿Da los resultados esperados (ensaye con varias entradas)?


RESPECTO A SUS PROPIAS HABILIDADES (usted debe autoevaluarse):

+ ¿Sabe usted usar CLOUD 9 (entrar, crear un directorio, navegar por los directorios, ver un archivo, crear un archivo y salvarlo)?
+ ¿Sabe usted usar un terminal Linux y el editor gedit (entrar, crear un directorio, navegar por los directorios, ver un archivo, crear un archivo y salvarlo)?
+ ¿Sabe usted editar un programa?
+ ¿Sabe usted compilar un programa?
+ ¿Sabe usted ejecutar un programa?
+ ¿Sabe usted depurar un programa?
